# Autoresearch 학습 시간/데이터 전략 가이드

## 1. Autoresearch의 핵심 원칙: "Epoch이 아니라 시간으로 제한한다"

```
❌ "10 epoch 돌려라"
✅ "5분 동안만 학습해라" (wall clock 기준)
```

Karpathy가 이렇게 설계한 이유:
- **공정한 비교**: 모델 크기를 키우면 epoch당 시간이 늘어남 → epoch 수가 같아도 불공정. 5분이면 어떤 실험이든 동일 조건
- **자동화에 적합**: AI가 batch_size를 2배로 키우면 step 수는 절반이 되지만, 5분은 그대로이므로 자동 비교 가능
- **GPU 활용 극대화**: 5분 × 12실험/시간 × 8시간(수면) = **~100실험**

---

## 2. dual-distillation_ver2에서의 현실적 문제

현재 프로젝트는 **epoch 기반** (`train.epochs=10`)이고, 데이터셋 크기에 따라 학습 시간이 크게 달라집니다:

| 상황 | 예상 시간 (A100 기준) |
|------|---------------------|
| COCO만 (~118K장), 10 epoch | ~30분~1시간 |
| COCO + Flickr30K + WIT, 10 epoch | 수 시간 |
| batch_size=128, 10 epoch | 기본값 |
| batch_size=32, 10 epoch | 4배 더 오래 |

**10 epoch 풀 학습을 매 실험마다 돌리면?**
→ 1시간/실험 × 8시간 = **겨우 8실험** 😥

## 3. 효율적 활용을 위한 3가지 전략

### 전략 1: 시간 예산 제한 (Autoresearch 원본 방식)

```python
# train.py에 시간 제한 추가
TIME_BUDGET = 300  # 5분

start = time.time()
for epoch in range(epochs):
    for step, batch in enumerate(train_loader):
        if time.time() - start >= TIME_BUDGET:
            break  # 시간 다 되면 중단
        # ... 학습 ...
    if time.time() - start >= TIME_BUDGET:
        break
# → 평가 실행 → 결과 출력
```

**장점**: 실험 수 극대화 (5분 × ~100실험/하룻밤)
**단점**: 5분으로는 수렴이 부족할 수 있음 → 하이퍼파라미터 탐색에는 좋지만, 최종 성능 비교에는 부족

### 전략 2: 축소 데이터셋 + 짧은 epoch (추천 🌟)

```bash
# 데이터를 10%만 사용하여 빠르게 트렌드 파악
python scripts/preprocess/parse_lpcvc_sources.py \
  --data_root D:\LPCVC_Data \
  --out_dir D:\LPCVC_Data\prepared_jsonl_subset \
  --val_ratio 0.01 \
  --train_ratio 0.1   # ← 10%만 사용

# 3 epoch만 학습 (5~10분 내에 종료)
python run_train.py --config config.yaml \
  --override train.epochs=3 \
  --override data.train_jsonl=D:/LPCVC_Data/prepared_jsonl_subset/train.jsonl
```

**핵심 인사이트**: 전체 데이터 10 epoch에서 A가 B보다 좋으면, 10% 데이터 3 epoch에서도 **대부분 A가 더 좋습니다**. 완벽하진 않지만 80~90% 정도 순위 상관관계가 유지됩니다.

| 방식 | 실험당 시간 | 하룻밤 실험 수 | 신뢰도 |
|------|-----------|------------|--------|
| 풀 데이터 10 epoch | ~1시간 | ~8개 | ★★★★★ |
| 풀 데이터 3 epoch | ~20분 | ~24개 | ★★★★☆ |
| 10% 데이터 3 epoch | ~5분 | ~100개 | ★★★☆☆ |
| 10% 데이터 1 epoch | ~2분 | ~200개 | ★★☆☆☆ |

### 전략 3: 2단계 파이프라인 (가장 효율적 🏆)

```
🌙 Phase 1 (밤새): 축소 데이터 + 짧은 학습으로 대량 스크리닝
   └─ 10% 데이터, 3 epoch, ~5분/실험
   └─ 100개 실험 → top 10 후보 선별

☀️ Phase 2 (낮): 상위 후보만 풀 학습으로 검증
   └─ 100% 데이터, 10 epoch, ~1시간/실험
   └─ 10개 실험 → 최종 1개 선택
```

이 방식은 논문에서도 자주 쓰는 패턴입니다. NAS(Neural Architecture Search)에서도 "proxy task"라고 부릅니다.

## 4. program.md에 반영하는 방법

```markdown
## 실험 모드

### 탐색 모드 (기본)
- 데이터: 10% subset의 train.jsonl
- 학습: 3 epoch (또는 5분 시간 제한)
- 목적: 하이퍼파라미터/손실함수 조합 빠르게 탐색
- 실행: python run_train.py --config config.yaml \
        --override train.epochs=3 \
        --override data.train_jsonl=dataset/prepared_jsonl_subset/train.jsonl

### 검증 모드 (탐색에서 top 결과 확인 시)
- 데이터: 100% 전체 데이터
- 학습: 10 epoch
- 목적: 실제 성능 확인
- 조건: 탐색 모드에서 val_mean_r10이 기준선 대비 3%p 이상 개선된 경우만
```

## 5. 요약

| 질문 | 답변 |
|------|------|
| 몇 epoch? | 탐색은 **3 epoch**, 검증은 **10 epoch** |
| 데이터 얼마나? | 탐색은 **10% subset**, 검증은 **전체** |
| 실험당 시간? | **5~10분** (탐색), **1시간** (검증) |
| 하룻밤 실험 수? | **~100개** (탐색), **~8개** (검증) |
| 신뢰도? | 축소 학습 결과의 **80~90%가 풀 학습에서도 유지** |
