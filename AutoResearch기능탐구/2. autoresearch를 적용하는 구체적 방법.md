# Autoresearch를 dual-distillation_ver2에 적용하는 방법

## 1. 프로젝트 구조 파악 — Autoresearch 관점 매핑

Autoresearch는 **3파일 원칙** (`prepare.py` 고정 / `train.py` AI수정 / `program.md` 사람작성)으로 설계되었습니다.
dual-distillation_ver2는 다중 파일 구조이므로 **어떤 파일이 어떤 역할에 해당하는지** 매핑이 핵심입니다.

### 파일 역할 분류

```
🔒 고정 (prepare.py 역할) — AI가 절대 수정하면 안 되는 파일
├── src/lpcvc_retrieval/data.py        # 데이터 파이프라인, 로더
├── src/lpcvc_retrieval/metrics.py     # Recall@K 평가 지표
├── src/lpcvc_retrieval/config.py      # 설정 로딩/파싱
├── src/lpcvc_retrieval/logger.py      # WandB 로거
├── src/lpcvc_retrieval/export.py      # ONNX export 유틸
├── src/lpcvc_retrieval/ema.py         # EMA 구현 (안정적)
├── scripts/eval.py                    # 평가 스크립트
├── scripts/export_onnx_split.py       # ONNX export
├── scripts/extract_features.py        # Teacher feature 추출
├── run_train.py                       # 엔트리포인트 (얇은 래퍼)
└── scripts/preprocess/               # 데이터 전처리

🤖 AI 수정 대상 (train.py 역할, 아래 표 참조)
├── config.yaml                        # 하이퍼파라미터/설정
├── src/lpcvc_retrieval/train.py       # 학습 루프 + 손실 조합
├── src/lpcvc_retrieval/losses.py      # 손실 함수 모음
├── src/lpcvc_retrieval/distill.py     # Teacher + 증류 로직
├── src/lpcvc_retrieval/dual_tower.py  # Student 모델 구조
└── src/lpcvc_retrieval/model.py       # 모델 팩토리

👤 사람 작성 (program.md 역할)
└── program.md                         # [새로 생성] AI 에이전트 지시서
```

### 왜 단일 파일이 아닌 다중 파일을 수정 대상으로 두는가?

Autoresearch 원본은 GPT 사전학습이라는 **단일 태스크**이므로 `train.py` 하나로 충분했지만,
이 프로젝트는 **구조적 복잡도가 높아** 수정 대상을 여러 파일로 나눠야 효과적입니다.

| 수정 대상 파일 | AI가 건드릴 수 있는 영역 | 위험도 |
|---|---|---|
| `config.yaml` | 하이퍼파라미터 (LR, batch, loss weight, temp 등) | 🟢 낮음 |
| `train.py` | 학습 루프, 손실 조합 비율, 스케줄러 | 🟡 중간 |
| `losses.py` | 손실 함수 추가/수정, 가중치 전략 | 🟡 중간 |
| `distill.py` | Teacher 설정, adaptive weighting 로직 | 🟠 높음 |
| `dual_tower.py` | Student 모델 아키텍처 변경 | 🔴 매우높음 |
| `model.py` | 모델 팩토리 (구조변경 시만) | 🔴 매우높음 |

> [!TIP]
> **실전 전략**: 처음엔 `config.yaml`만 AI에게 수정 허용 → 안정화 후 `losses.py`/`train.py` → 최종적으로 `dual_tower.py` 순서로 확대

## 2. Autoresearch 적용을 위한 프로젝트 재구성

### 2.1 핵심 변경: 시간 예산 기반 학습으로 전환

현재 코드는 epoch 기반(`train.epochs=10`)이지만, autoresearch는 **고정 시간 예산**(5분)이 핵심입니다.
`train.py`에 시간 예산 로직을 추가해야 합니다.

```python
# src/lpcvc_retrieval/train.py 수정 예시
import time

TIME_BUDGET_SECONDS = 300  # 5분 고정

# 기존 epoch 루프를 시간 기반으로 변경
start_time = time.time()
epoch = 0
while True:
    elapsed = time.time() - start_time
    if elapsed >= TIME_BUDGET_SECONDS:
        break
    epoch += 1
    # ... 기존 epoch 학습 코드 ...
    
    # 매 step 후에도 시간 체크
    for step, batch_data in enumerate(pbar):
        if time.time() - start_time >= TIME_BUDGET_SECONDS:
            break
        # ... 기존 step 코드 ...
```

## 4. Single Teacher 전환 시 autoresearch가 탐색할 구체적 실험들

현재 Dual Teacher (SigLIP2 + PE-Core) → Single Teacher로 전환 계획을 반영한 실험 시나리오:

### 4.1 config.yaml 수준의 빠른 전환

```yaml
# 방법 A: SigLIP2만 사용
distill:
  use_teacher: true
  teachers:
    - name: ViT-gopt-16-SigLIP2-256
      pretrained: webli
      input_size: 256
  adaptive_teacher_weight: false  # teacher 1개이므로 불필요
  teacher_weight_mode: static

# 방법 B: PE-Core만 사용
distill:
  use_teacher: true
  teachers:
    - name: PE-Core-bigG-14-448
      pretrained: meta
      input_size: 448
  adaptive_teacher_weight: false
  teacher_weight_mode: static
```

> [!IMPORTANT]
> Single Teacher에서는 `distill.py`의 `create_teacher()`가 자동으로 `OpenClipTeacher`(단일)를 반환합니다 — `EnsembleTeacher` 래퍼 없이. 기존 코드가 이미 이 분기를 지원하므로 **config.yaml만 바꾸면 작동**합니다.

### 4.2 distill.py의 코드 단순화

Single Teacher로 확정되면 distill.py에서 unnecessary한 앙상블 코드를 제거할 수 있습니다:

```diff
 # distill.py의 compute_affinity_distill_loss에서
 # Ensemble 분기 전체를 제거하고 Single Teacher 분기만 남김
-    if isinstance(teacher_output, list):
-        # ... 100줄의 앙상블 로직 ...
-    else:
         # Single Teacher Case
         t_img, t_txt = teacher_output
         t_sim = t_img @ t_txt.t()
         # ... 단순한 로직 유지 ...
```

### 4.3 실험 시나리오 예시

```
실험 1: 기준선 (현재 Dual Teacher, config 그대로)
  → val_mean_r10: 0.45 ✅ keep (baseline)

실험 2: SigLIP2만 사용
  → val_mean_r10: 0.44 → -0.01 ❌ discard (미세 하락)

실험 3: PE-Core만 사용
  → val_mean_r10: 0.43 → -0.02 ❌ discard

실험 4: SigLIP2만 + affinity_temp 0.07 → 0.05
  → val_mean_r10: 0.46 → +0.01 ✅ keep (Single이 더 나음!)

실험 5: w_distill_affinity 0.8 → 1.2
  → val_mean_r10: 0.47 → +0.01 ✅ keep

실험 6: w_hard_negative 0.1 → 0.2
  → val_mean_r10: 0.48 → +0.01 ✅ keep
...
```

## 6. 핵심 파일별 AI가 탐색 가능한 변경 사항 요약

### [config.yaml](https://github.com/lpcvc-2026-CNU/dual-distillation_ver2/blob/main/config.yaml)

| 설정 키 | 현재값 | 탐색 범위 | 영향 |
|---------|--------|----------|------|
| `train.lr` | 5e-4 | 1e-4 ~ 1e-3 | 수렴 속도/안정성 |
| `train.weight_decay` | 0.05 | 0.01 ~ 0.1 | 과적합 방지 |
| `train.warmup_epochs` | 3.0 | 1.0 ~ 5.0 | 초기 학습 안정성 |
| `loss.w_contrastive` | 1.0 | 0.5 ~ 2.0 | contrastive vs distill 비율 |
| `loss.w_distill_affinity` | 0.8 | 0.3 ~ 1.5 | 증류 강도 |
| `loss.w_hard_negative` | 0.1 | 0.0 ~ 0.3 | Hard neg 학습 강도 |
| `loss.w_text_text` | 0.05 | 0.0 ~ 0.2 | 텍스트 정렬 강도 |
| `distill.affinity_temp_start` | 0.12 | 0.05 ~ 0.2 | 초기 증류 부드러움 |
| `distill.affinity_temp_end` | 0.07 | 0.03 ~ 0.15 | 후기 증류 선명도 |
| `distill.distill_margin_thr` | 0.2 | 0.1 ~ 0.5 | Selective distill 임계값 |
| `model.embed_dim` | 256 | 128 / 256 / 512 | 임베딩 차원 (속도 vs 성능) |

### [losses.py](https://github.com/lpcvc-2026-CNU/dual-distillation_ver2/blob/main/src/lpcvc_retrieval/losses.py)

현재 구현된 손실 함수 4개 + AI가 추가 시도 가능한 것들:

| 현재 구현 | 역할 |
|----------|------|
| `siglip_loss` | 주 contrastive (sigmoid 기반, 1:5 COCO 지원) |
| `pairwise_ranking_loss` | Hard negative ranking |
| `hard_negative_contrastive_loss` | Top-K hard neg에 집중 학습 |
| `text_text_contrastive_loss` | 같은 이미지 캡션들을 가깝게 |

| AI가 추가 시도 가능 | 기대 효과 |
|-------------------|----------|
| CRD Loss | Student-Teacher 표현 구조 정렬 |
| PKD (Patient KD) | Teacher 중간 레이어 지식 전달 |
| Cosine Similarity Loss | 직접적 임베딩 방향 정렬 |
| InfoNCE 변형 | Temperature 스케줄과 함께 |

### [distill.py](https://github.com/lpcvc-2026-CNU/dual-distillation_ver2/blob/main/src/lpcvc_retrieval/distill.py)

Single Teacher 전환 시 핵심 변경 포인트:
- `teachers` 리스트를 1개로 → `create_teacher()`가 `OpenClipTeacher` 직접 반환
- `adaptive_teacher_weight` → `false`로 (teacher 1개면 불필요)
- `compute_affinity_distill_loss()`의 Single Teacher 분기만 사용됨
- `affinity_kl_rows()`의 `temp` 파라미터가 핵심 조정 대상

### [dual_tower.py](https://github.com/lpcvc-2026-CNU/dual-distillation_ver2/blob/main/src/lpcvc_retrieval/dual_tower.py)

| 현재 구조 | AI가 시도 가능한 변형 |
|----------|-------------------|
| Image: `mobilenetv4_hybrid_large` | `mobilenetv4_hybrid_medium`, `efficientnet_b0` |
| Text: `DatologyAI/retr-opt-vit-b-32` | 다른 OpenCLIP text tower |
| Projection: `nn.Linear` (1층) | `nn.Sequential(Linear, GELU, Linear)` (2층 MLP) |
| embed_dim: 256 | 128 (더 작고 빠른 추론) |

> [!CAUTION]
> Image/Text tower 자체를 바꾸면 **사전학습 가중치가 달라지므로** 결과 변동이 크고 ONNX export에도 영향을 줍니다. 이 변경은 **다른 모든 하이퍼파라미터가 충분히 탐색된 후**에만 시도하세요.

## 7. Offline Feature 모드를 활용한 고속 실험

Teacher를 매번 실행하면 VRAM과 시간을 많이 소모합니다. **Offline Feature 모드**를 활용하면 autoresearch 실험 속도를 크게 높일 수 있습니다.

```bash
# 1회만 실행: Teacher 임베딩 추출 (수십 분 소요)
python scripts/extract_features.py --config config.yaml --out_dir features

# 이후 모든 실험에서 재사용 (Teacher VRAM 0, 속도 2-3배 향상)
python run_train.py --config config.yaml \
  --override distill.offline_feature_dir=features
```

이 방식은 autoresearch의 5분 실험 루프와 매우 잘 맞습니다:
- Teacher 로딩/추론 시간 제거 → **같은 5분에 더 많은 step 학습**
- VRAM 절감 → **더 큰 batch_size 가능**
- 재현성 확보 → **Teacher 출력이 고정되므로 순수하게 Student 변경만 비교**
